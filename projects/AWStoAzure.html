AWS to Azure migration

Technologies: Decom of AWS tenant and move workspaces to Azure by creating all new Azure VDI machines

Overview

Detailed description of the project, including objectives, challenges faced, and solutions implemented.

Key Features

  • Decom entire AWS tenant including workspaces, E2C instances, and networking components
  • Created subnets, Azure VDI, and Golden image in Azure tenant
  • Granted permission to auditors and users for access and streamlined message

Outcome

The project successfully allowed auditors to maintain access to machines while saving cost on not having to spend money on AWS tenant.
